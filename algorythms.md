# Конспект по дисциплине Алгоритмы и структуры данных

## Тема: Базовые структуры данных

Физическая структура данных - это отображение способа представления данных в памяти компьютера.

Любая структура данных может быть описана на 3-х уровнях:
1. Функциональная спецификация  - Указывает для некоторого класса имён операции и их свойства, которые ему разрешены. 
2. Логическое описание          - Задаёт декомпозицию объектов и некоторых операций на более элементарные
3. Физическое представление     - Задаёт метод размещения в памяти компьютера величин, которые входят в структуру объекта,
    взаимосвязи (взаимоотношения) восле ними, и способ кодирования некоторой операции языком программирования.

Различают простые и интегрированные структуры данных.
- Простые           - Нельзя разделить на составляющие;
- Интегрированные   - Состоят из простых;

Основные операции над структурами данных
- Создание      - Выделение памяти;
- Уничтожение   - Освобождение памяти;
- Выбор         - Доступ к данным;
- Возобновление - Изменение данных;

### Простые структуры данных:
1. Арифметические типы
2. Перечисления
3. Указатели

### Статические структуры данных

#### Массивы

Массив - структура данных, которая характеризуется *(логическая структура)*:
1. Фиксированным набором элементов одного типа;
2. Каждый элемент имеет уникальный набор значений индексов;
3. Количество индексов определяют размерность
4. Обращение к каждому элементу массива совершается по имени массива
    и значению индекса элемента;

Физическая структура массива - это способ размещения элементов массива в памяти компьютера;

Осноные операции с массивами:
- Сортировка
- Поиск

#### Разрежженные массивы

Разрежженные массивы - Массивы, большинство элементов которых одинаковые, 
    поэтому, в памяти сохраняются только отличающиеся значения;

Существует 2 типа разрежженых массивов:
1. В которых размещение отличающихся элементов описываются математически
2. Массивы со случайным размещением элементов

#### Множество

Множество - Массив с уникальными значениями *(без повторов)*, порядок не важен.

Пустое множество называется **нулевым**;

Множество может быть подмножеством другого множества,
    если все элементы первого множества входят в состав второго.

Надмножество - наоборот;

Основные операции над множествами:
1. Объединение
2. Пересечение
3. Разница
4. Симетрическая разница
5. Проверка на принадлежность элемента


#### Структуры

Совокупность элементов разных типов

Основные операции:
1. Квалификация - Операция доступа к полю

#### Битовые типы

Используются для компактного размещения данных.

Для них используются операции буллевой алгебры:
- ! (NOT)
- | (OR)

- ^ (XOR)

Особенности полустатических структур:
1. Изменяемая длина;
2. Изменение длины структуры происходит в определенных пределах;

#### Стеки (LIFO)

Стек - множество изменяемого количества данных,
    над которым производятся следующие операции *(функциональное описание)*:
1. Пополнение стека;
2. Проверка на пустоту стека;
3. Просмотр последних данных;
4. Уничтожение последних данных;

Логическое описание стека: 

Дополнительные операции:
1. Определение количества элементов;
2. Очищение стека;
3. Чтение с вершины (pop -> push);

При представлении стека в статической памяти, для него выделяется такая же память, как и для вектора.

В описании этого вектора, кроме обычных параметров, находится и указатель стека (адрес вершины).

Размер любого стека ограничен размерами вектора.

Указатель стека может указывать или на первый свободный элемент стека, или на последний записаный.

При внесении элемента в стек, он записывается на место, на которое показывает указатель.

После этого, указатель модифицируется таким образом, чтобы он укахывал на следующих свободный элемент.

#### Очередь (FIFO)

Очередь - это множество данных, переменного количества, над которой можно выполнять следующие операции *(функциональное описание)*:
1. Пополнение новыми данными
2. Проверка на пустоту
3. Просмотр первых добавленных элементов
4. Уничтожение первых добавленных элементов

*Логическое описание очереди:*
Очередью (**FIFO**) называется такой последовательный список с переменной длиной,
    в котором добавление элементов пыполняется с боку хвоста очереди (с конца), а удаление - с начала.

Основные операции такие же, как и у стека.

Приоритет:
- Абсолютный (прерывание)
- Относительный (следующий в очереди)

##### Деки

Дек - особый вид очереди: очередь с двумя концами.

Добавление и удаление элементов может происходить с любого из концов.

Вместо головы и хвоста говорят про левый и правый концы списка.

Основные операции:
1. Добавление элементов
2. Удаление элементов
3. Определения размера (количество элементов)
4. Очищение

Физическая структура дека в статической памяти идентическая структуре **кольцевой** очереди.

#### Линейные списки

Линейные списки являются обобщением предыдущих структур.
Позволяют представить множество таким образом, чтобы каждый элемент был доступным и при этом не было бы надобности трогать остальные.

Количество элементов называется длиной списка.

В списках, в отличии от множеств, элементы могут повторяться.

Частые операции над линейными списками:
1. Добавление       - Добавляет элемент в конкретное место списка, перемещая при этом другие элементы.
2. Локализация      - Как результат, возвращает первую позицию элемента в списке.
3. Выбор элемента   - Возвращает элемент в нужной позиции списка, или undefined, если такой элемента нет.
4. Удаление         - Удаляет элемент, смещая соседние
5. Выбор prev/next  -
6. Очистка          - Удаляет все элементы списка.

Методы хранения линейных списков:
- Последовательное хранение
- Связанное хранение

#### Мультисписки

Мультисписок - многосвязный список, каждый элемент которого может одновременно входить в несколько односвязных списков.

Плюсы мультисписков:
1. Экономия памяти
2. Обеспечение целостности данных

Специфика мультисписка только в операции удаления: Удаление элемента из одного списка не означает необходимость удаления его из памяти, поскольку он может находится в других списках.
Память освобождается только тогда, когда элемент не входит ни в один из подсписков.

### Строка

Строка - это линейно упорядоченная последовательность символов, которые входят в конечное множество символов - алфавит.

Свойства:
- Длина может быть переменной, хотя алфавит фиксированный.
- Обращение к символам строки происходит с одной стороны последовательности, в основном, с начала.

Базовые операции над строками:
- Определение длины строки.
- Объединение нескольких строк в одну.
- Выделение подстроки. Выделяет последовательность символов, начиная с заданной позиции заданной длины.
- Сревнение строк. Сравниваются первые символы двух строк. По первому отличающемся символу и его дальности по алфавиту (><)
- Поиск вхождения символа. Возвращает позицию первого вхождения или null (-1)

Строки можно представлять в виде списоков.

### Динамические структуры данных

Динамическая структура данных характеризируется отсутствием физической смежности элементов структуры в памяти, нестойкостью и непредсказуемостью структуры в процессе её обработки.

Для установки связи между элементами динамической структуры, используются указатели, с помощью которых устанавливаются явные связи между элементами.

Элемент динамической структуры состоит из двух полей:
- Информационное поле (поле данных)
- Поле связи

Преимущества связанного представления данных:
1. Возможность изменения структуры - Необходимо лишь изменить укзатели.
2. Размер структуры ограничивается только объёмом памяти

Недостатки:
1. Требуется высокая квалификация программиста.
2. Дополнительная память на хранение указателей.
3. Большие затраты времени на выполнение программы.

### Графы

Граф - сложная нелинейная многосвязная динамическая структура, которая отображает свойства и связи сложного объекта.

Основные свойства графов:
1. На каждый элемент может быть неограниченное количество ссылок.
2. Каждый элемент может быть связан с любым количеством элементов.
3. Каждая связь может иметь направление и вес

В узлах графа хранится информация про элеменгты объекта. Связи между узлами задаются рёбрами объекта.

Рёбра графа могут быть направлены, тогда граф называется ориентированным.

Методы представления графа в памяти компьютера:
1. Матричный
2. Связанный нелинейными списками

Вид метода зависит от природы данных и операций над ними.

Если задача требует большое количество включений и выключений узлов, то логичнее представить граф в виде связанных списков, 
иначе можно использовать матричное представление.

Используя матрицу смежности, её элементы располагаются в памяти компьютера элементами массива.
Для простого графа, матрица состоит из 0 и 1, для мультиграфа - из 0 и целых чисел, которые показывают кратность рёбер, а для взвешенного графа - с нулей и действительных чисел, которые задают вес каждого ребра.
